# TDD 測試過程及結果說明 - 題目 299 (UVA 299)

## 題目概述
UVA 299 是一個火車車廂排序問題，需要計算將車廂排列成 1,2,3,... 順序所需的最少相鄰交換次數。這實際上是計算排列中的逆序對數量。

## TDD (測試驅動開發) 策略

### TDD 三階段流程
1. **紅燈階段 (Red)**：先寫測試，讓測試失敗
2. **綠燈階段 (Green)**：實現功能，讓測試通過
3. **重構階段 (Refactor)**：優化代碼，保持測試通過

### 測試案例設計原則
- **邊界測試**：測試空列表、單元素、已排序情況
- **基本交換測試**：測試只需要一次交換的情況
- **複雜排列測試**：測試需要多次交換的複雜排列
- **極端情況測試**：測試最大長度 L=50 的反向排序

## 實現細節

### 核心演算法
```python
def count_swaps(train):
    if not train:
        return 0
    
    cars = train[:]  # 複製避免修改原列表
    swaps = 0
    
    n = len(cars)
    for i in range(n):
        for j in range(0, n-i-1):
            if cars[j] > cars[j+1]:
                cars[j], cars[j+1] = cars[j+1], cars[j]
                swaps += 1
    
    return swaps
```

### 關鍵設計決策
1. **使用氣泡排序**：直接計算交換次數，等於逆序對數量
2. **列表複製**：避免修改輸入參數
3. **邊界處理**：正確處理空列表情況
4. **簡單實現**：使用雙重循環，易於理解和驗證
