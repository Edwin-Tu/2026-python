# TDD 測試過程及結果說明 - 題目 118 (UVA 118)

## 題目概述
UVA 118 是一個機器人移動模擬問題，需要處理機器人在矩形土地上的移動、轉向，以及邊界檢查和標記機制。

## TDD (測試驅動開發) 策略

### TDD 三階段流程
1. **紅燈階段 (Red)**：先寫測試，讓測試失敗
2. **綠燈階段 (Green)**：實現功能，讓測試通過
3. **重構階段 (Refactor)**：優化代碼，保持測試通過

### 測試案例設計原則
- **單元測試**：每個測試方法測試一個特定的功能點
- **邊界測試**：測試極端情況（如邊界掉落）
- **正常流程測試**：測試基本功能
- **異常情況測試**：測試標記機制和錯誤處理

## 測試案例詳解

### 1. 機器人初始化測試
```python
def test_robot_initialization(self):
    """測試機器人初始化位置和方向"""
```
- **目的**：確保機器人能正確初始化位置和方向
- **測試數據**：位置 (1,2)，方向 'N'
- **預期結果**：x=1, y=2, direction='N', lost=False

### 2. 轉向測試
```python
def test_turn_left(self):
    """測試左轉指令 L"""
def test_turn_right(self):
    """測試右轉指令 R"""
```
- **目的**：驗證轉向邏輯正確性
- **測試流程**：連續執行 4 次轉向，應該回到原方向
- **預期結果**：N → W → S → E → N (左轉)
- **預期結果**：N → E → S → W → N (右轉)

### 3. 前進測試
```python
def test_move_forward_north(self):
    """測試向前移動 F，面向北方"""
# 類似測試東、南、西方向
```
- **目的**：驗證不同方向的前進邏輯
- **測試數據**：各方向的起始位置和預期終止位置
- **預期結果**：正確的座標變化

### 4. 邊界掉落測試
```python
def test_robot_falls_off_north_boundary(self):
    """測試機器人從北邊界掉落"""
# 類似測試南、東、西邊界
```
- **目的**：測試邊界檢查和掉落機制
- **測試流程**：將機器人放在邊界，執行前進指令
- **預期結果**：lost=True，位置保持在掉落前，標記被留下

### 5. 標記機制測試
```python
def test_scent_mark_prevents_fall(self):
    """測試標記防止機器人掉落"""
```
- **目的**：驗證標記能防止後續機器人掉落
- **測試流程**：
  1. 第一個機器人掉落，留下標記
  2. 第二個機器人在同位置嘗試前進
- **預期結果**：第二個機器人忽略前進指令，不掉落

### 6. 指令序列測試
```python
def test_execute_instruction_sequence(self):
    """測試執行一連串指令"""
```
- **目的**：測試複雜指令序列的執行
- **測試數據**：指令序列 "LFRF"
- **預期結果**：正確的最終位置和方向

### 7. 輸出格式測試
```python
def test_robot_output_format(self):
    """測試機器人輸出格式"""
```
- **目的**：確保輸出格式符合題目要求
- **測試數據**：正常狀態和 LOST 狀態
- **預期結果**：正確的字串格式

### 8. 多機器人測試
```python
def test_multiple_robots(self):
    """測試多個機器人依序動作"""
```
- **目的**：驗證多個機器人能正確依序執行
- **測試流程**：兩個機器人依次執行指令
- **預期結果**：各自的狀態正確且不互相干擾
