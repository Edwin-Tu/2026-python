==================================================
二、測試設計
==================================================

測試檔案：test_10035.py

本題共設計 8 個測試，分為兩大類：

（一）核心邏輯測試：count_carries()

1. test_no_carry
   測試沒有進位的情況
   範例：123 + 456
   預期：0 次進位

2. test_single_carry
   測試剛好發生一次進位
   範例：123 + 594
   預期：1 次進位

3. test_multiple_carries
   測試多次連續進位
   範例：555 + 555
   預期：3 次進位

4. test_different_length
   測試不同位數的整數相加
   範例：1 + 999
   預期：3 次進位

5. test_with_zero
   測試 0 與 0 相加
   範例：0 + 0
   預期：0 次進位


（二）整合測試：solve()

6. test_solve_basic
   測試多筆輸入與基本輸出格式是否正確
   包含：
   - 無進位
   - 多次進位
   - 單次進位
   - 終止條件 0 0

7. test_solve_with_single_carry_format
   測試單次進位時的英文輸出格式是否正確
   預期輸出：
   1 carry operation.

8. test_solve_multiple_lines
   測試多筆皆無進位時的輸出結果
   預期每一行都輸出：
   No carry operation.

三、TDD 開發過程
==================================================

【Phase 1：Red】

先閱讀題目後，確認核心規則為：
- 由右至左模擬直式加法
- 若某位總和大於等於 10，則產生一次進位
- 需累計所有進位次數
- 當輸入為 0 0 時，停止處理

接著先撰寫測試，不先寫完整主程式。
此時由於 count_carries() 與 solve() 尚未完成，測試會失敗。

這個階段的目的：
- 先把題目需求轉成具體可驗證的測試
- 避免一開始就把程式邏輯寫死


【Phase 2：Green】

先實作 count_carries(a, b)：

做法：
1. 用兩個索引從字串最後一位開始
2. 每次取出目前位數
3. 加上上一位留下來的 carry
4. 若總和 >= 10，代表本位發生進位
5. count 加 1，carry 設為 1
6. 若總和 < 10，carry 設為 0
7. 直到兩數所有位數都處理完畢

完成後，核心邏輯測試通過：
- 無進位
- 單次進位
- 多次進位
- 不同位數
- 0 與 0

再實作 format_result(carries)：
- 0 次進位 → No carry operation.
- 1 次進位 → 1 carry operation.
- 大於 1 次 → X carry operations.

最後實作 solve(input_data)：
1. 逐行讀取輸入
2. 去除空白行
3. 拆出兩個數字
4. 若讀到 0 0，停止
5. 呼叫 count_carries()
6. 呼叫 format_result()
7. 將結果組合成多行輸出


【Phase 3：Refactor】

在全部測試通過後，進行程式整理與優化：

1. 補上完整繁體中文註解
   - 說明函式用途
   - 說明參數與回傳值
   - 說明每個步驟的邏輯

2. 函式分層
   - count_carries()：只負責計算進位次數
   - format_result()：只負責輸出文字格式
   - solve()：只負責輸入解析與整體流程

3. 改善變數命名
   - carry：表示目前是否有進位
   - count：表示總進位次數
   - digit_a / digit_b：表示當前位數

4. 增加空行處理
   - 避免輸入中有空白列時造成錯誤

==================================================
五、程式設計說明
==================================================

本題的關鍵在於模擬「直式加法」過程，而不是直接將兩數相加後再分析。

原因如下：
1. 題目要的是「進位次數」
2. 進位是一個逐位處理的過程
3. 從右往左模擬最符合題目原意

因此採用字串處理方式：
- 方便直接從個位數開始
- 位數不同時可以補 0
- 邏輯清楚且容易對應測試
