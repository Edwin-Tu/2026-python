二、測試檔案
==================================================

測試檔：test_10035.py

本題共設計 8 個測試，分為兩類：

（一）核心邏輯測試：count_carries()
1. test_no_carry
   測試沒有進位的情況
   範例：123 + 456 → 0 次進位

2. test_single_carry
   測試剛好一次進位
   範例：123 + 594 → 1 次進位

3. test_multiple_carries
   測試多次連續進位
   範例：555 + 555 → 3 次進位

4. test_different_length
   測試不同位數整數相加
   範例：1 + 999 → 3 次進位

5. test_with_zero
   測試 0 與 0 相加
   範例：0 + 0 → 0 次進位

（二）整合測試：solve()
6. test_solve_basic
   測試多筆輸入與基本輸出格式

7. test_solve_with_single_carry_format
   測試單次進位時的英文輸出格式
   預期：1 carry operation.

8. test_solve_multiple_lines
   測試多筆皆無進位時的輸出

【Phase 2：Green】

- 由右往左模擬直式加法
- 記錄目前是否有進位

實作後，可先通過以下測試：

- 單次進位

- 0 次進位 → No carry operation.
- 1 次進位 → 1 carry operation.

最後實作 solve(input_data)：
- 逐行讀取輸入
- 遇到 "0 0" 停止

==================================================
五、程式設計說明
==================================================

本題核心概念為模擬小學直式加法：

1. 從個位數開始處理
2. 每次將兩位數字與上一位進位相加
3. 若總和大於等於 10，代表產生進位
4. 累計進位次數
5. 持續處理直到所有位數都算完

此方法不需要真的做整數加法後再分析，而是直接依題意模擬加法過程，因此較直觀，也容易對應測試案例。
